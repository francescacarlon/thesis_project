Think of a noun phrase as a group of words that includes at least one noun, like 'three parties from Brooklyn'. These groups can function similarly, appearing in similar positions, such as before a verb, like 'three parties from Brooklyn arrive'. A commonly used formal system for representing this constituent structure in natural language is the context-free grammar (CFG). This system is made up of a set of rules or productions, which demonstrate how symbols in the language can be grouped and ordered, as well as a lexicon of words and symbols. For example, an NP (noun phrase) can be constructed from either a ProperNoun or a determiner (Det) followed by a Nominal, such as 'the flight'. These context-free rules can be hierarchically nested, like Det → a, Det → the, Noun → flight. This means that a determiner can be either 'a' or 'the', and a noun can be 'flight'. Using this CFG, we can generate a sequence of words, also known as a derivation. This series of rule expansions is represented as a parse tree, typically shown inverted with the root at the top. A CFG like that of L0 defines a formal language. Words strings that can be generated by a grammar are part of the formal language defined by that grammar and are referred to as grammatical sentences. Sentences that cannot be generated by a given formal grammar are not part of the language defined by that grammar and are referred to as ungrammatical. Just like we combine tokens or features in NLP tasks, words combine into meaningful units called constituents. Think of noun phrases (NPs) as a familiar example - they're word sequences built around nouns that function as a single unit in sentences. Just as we train models to recognize patterns, these constituents follow predictable patterns in language structure. Context-free grammars (CFGs) are formal systems for modeling these patterns, similar to how we use rule-based approaches in NLP. Like defining patterns for text processing, CFGs have two main components: production rules that specify how language elements combine, and a lexicon (think of it as a vocabulary mapping, similar to tokenization). The grammar effectively defines a formal language - strings that can be derived are "grammatical" (valid inputs), while those that can't are "ungrammatical" (invalid inputs), much like how we validate inputs in our NLP pipelines. This formal approach to modeling natural language is called generative grammar in linguistics.
